FAQ

제 사이트를 실제로 해킹하나요?

아니요. 무단 침투나 공격 대행이 아니라 소유자 승인 기반 코드·설정 리뷰입니다. Standard에서는 테스트 계정과 합의된 범위 안에서 권한·세션·파일 접근을 제한적으로 재현 확인할 수 있습니다.

침투테스트도 포함되나요?

광범위한 침투테스트는 기본 상품이 아닙니다. 필요한 경우 테스트 계정 기준의 제한적 수동 검증만 진행하며, 무단 스캔, 부하 테스트, 파괴적 테스트, 개인정보 다운로드는 제외합니다.

로그인이나 회원가입이 필요한가요?

필수는 아닙니다. 신청폼으로 의뢰 내용을 정리한 뒤 이메일로 접수합니다.

제출하면 바로 자동 점검이 시작되나요?

아니요. 먼저 권한과 점검 범위를 확인한 뒤 진행합니다. 무단 스캔이나 대량 요청은 하지 않습니다.

Google Sheets나 고객 대시보드는 없나요?

현재 접수는 이메일 중심으로 처리합니다. 반복 의뢰나 팀 단위 운영이 필요하면 동의 기반으로 Google Sheets, Notion 같은 기록 연동을 별도 협의합니다.

리포트는 어떤 형태로 받나요?

상담과 결제 후 Markdown 또는 PDF 형태로 전달합니다. 페이지에는 리포트 구성 예시만 공개하고, 실제 리포트는 의뢰 범위에 맞춰 개별 전달합니다.

의뢰 신청

입력 내용을 검증한 뒤 의뢰 메일로 바로 접수합니다. 접수 후 회신 이메일로 범위 확인과 결제 안내를 보냅니다.

입력한 정보는 상담 접수, 권한 확인, 점검 범위 협의 목적으로만 사용합니다. 민감한 비밀번호, API 키, 주민등록번호는 보내지 마세요.
이용약관 보기

운영 전 법률 전문가 검토가 필요한 초안입니다. 신청 전 핵심 범위와 책임 기준을 확인하기 위한 안내입니다.

  1. 서비스 목적 VibeSec Review는 고객이 소유하거나 정당한 권한을 가진 웹앱에 대해, 사전에 합의된 범위 안에서 보안 설정, 코드 구조, 접근 권한, 세션, 파일 접근 등 위험 요소를 점검하고 개선 방향을 제안합니다.
  2. 소유자 권한 보증 고객은 서비스 URL, 저장소, 테스트 계정, 설정 정보에 대해 점검을 의뢰할 정당한 권한이 있음을 보증합니다. 제3자 시스템을 무단으로 의뢰할 수 없습니다.
  3. 점검 범위 점검은 신청서와 별도 합의에서 승인된 범위로 제한됩니다. Basic은 공개 URL, 저장소, 설정 중심의 비침투형 1차 점검이며, Standard는 테스트 계정과 승인 범위 안에서 제한적 수동 재현 확인을 포함할 수 있습니다.
  4. 금지 행위 무단 침투, 무단 스캔, 부하 테스트, 파괴적 테스트, 개인정보 또는 영업비밀의 대량 조회·다운로드, 합의 범위 밖 테스트는 수행하지 않습니다.
  5. 고객 제공 정보 고객은 신청 정보와 제공 자료가 정확하도록 해야 합니다. 정보 누락이나 오류가 있으면 일정, 비용, 결과 범위가 변경될 수 있습니다.
  6. 접근정보 관리 테스트 계정, 저장소 초대, 임시 접근 권한은 점검 목적에만 사용합니다. 작업 종료 또는 고객 요청 시 접근 해제, 삭제, 권한 회수를 요청하거나 수행합니다.
  7. 결과물의 한계 리포트와 Fix PR 또는 패치 초안은 확인 가능한 범위의 위험 분석과 개선 제안입니다. 모든 취약점 발견, 보안 완전성, 무사고 운영, 법령 준수의 완전 보장을 의미하지 않습니다.
  8. Fix PR 반영 수정 PR이나 패치 초안의 실제 반영, 배포, 백업, 롤백, 운영 적용 여부는 고객이 최종 판단합니다.
  9. 일정·비용 확정 신청 접수만으로 계약이 확정되지 않습니다. 점검 범위와 난이도 확인 후 일정, 비용, 산출물을 안내하고 고객이 동의하면 진행합니다.
  10. 취소·환불 작업 시작 전 취소는 실제 발생 비용을 제외하고 환불할 수 있습니다. 작업 시작 후에는 수행된 분석, 재현, 문서화, PR 작성 등 진행분을 기준으로 환불 가능 금액을 산정합니다. 제공자 귀책으로 서비스를 제공할 수 없는 경우 미제공분을 환불합니다.
  11. 비밀유지 양 당사자는 비공개 코드, 설정, 취약점 정보, 접근정보, 사업상 정보를 외부에 공개하지 않습니다. 법령상 요구, 분쟁 대응, 고객 승인 시 필요한 범위에서 예외로 합니다.
  12. 책임의 제한 VibeSec Review는 고의 또는 중대한 과실에 따른 책임을 배제하지 않습니다. 다만 고객의 권한 없는 의뢰, 부정확한 정보 제공, 합의 범위 밖 요청, 고객의 임의 수정·배포, 제3자 서비스 장애 등 제공자 책임 없는 사유로 발생한 손해는 책임이 제한될 수 있습니다.

  7. 보유 기간 상담만 진행하고 의뢰가 성립하지 않은 접수 정보는 마지막 연락일로부터 90일 이내 파기합니다. 유료 의뢰로 전환된 기록은 거래 종료 후 5년, 불만·분쟁 처리 기록은 처리 완료 후 3년까지 보관할 수 있습니다.
  8. 파기 방법 보유 기간이 끝난 전자 파일, 메일, 접수 기록은 복구하기 어려운 방식으로 삭제합니다. 법령상 보존이 필요한 기록은 별도 분리해 접근을 제한합니다.
