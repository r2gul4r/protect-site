# AGENTS.md

## 프로젝트

- 이 workspace는 `VibeSec Review` MVP를 만든다.
- 목표는 결제형 SaaS가 아니라 문의형 랜딩페이지, 신청폼, 샘플 리포트, 수동 결제 안내, Codex 기반 리포트 자동화 초안이다.
- 최종 보안 판단과 고객 전달물은 사람이 검수한다.

## 기본 태도

- 한국어로 짧고 직접적으로 답한다.
- README를 제품 계약의 1차 소스로 본다.
- 보안 서비스 문구는 과장하지 말고, 확인된 사실과 가능성을 분리한다.
- 불법 침투, 공격 대행, 무단 스캔처럼 보이는 표현을 쓰지 않는다.

## 절대 경계

- 소유자 또는 관리 권한자의 명시적 승인 없는 대상은 점검하지 않는다.
- DDoS, 대량 요청, 계정 탈취, 데이터 삭제/변조, 개인정보 다운로드, 실제 결제 시도, 악성코드 업로드는 금지한다.
- 제3자 서비스 무단 테스트 요청은 거절한다.
- 공격 재현 절차는 고객이 수정에 필요한 수준까지만 적고 악용 가능한 상세 절차는 쓰지 않는다.

## MVP 범위

- 1순위는 `/`, `/pricing`, `/scope`, `/apply`, `/sample-report`, `/faq`, `/terms`, `/privacy`다.
- 신청폼은 권한 확인과 점검 범위 동의를 반드시 받는다.
- 결제는 초기에는 계좌이체, PayPal, 플랫폼 에스크로 안내까지만 다룬다.
- 카드결제, 고객 대시보드, 자동 스캐너 SaaS, 무인 리포트 발송은 MVP 밖이다.

## 보안 리뷰 상품

- Basic Review는 코드, 설정, 헤더, 의존성, secret 노출, 공개 repo 중심의 비침투형 점검이다.
- Standard Review는 테스트 계정과 스테이징 등 명시 승인된 범위 안에서 인증/권한/API 흐름을 본다.
- Fix PR Add-on은 고객 승인 후 패치 또는 PR 초안을 제공하는 별도 옵션이다.

## 구현 원칙

- 기본 스택은 Next.js와 Tailwind CSS를 우선 고려한다.
- DB 없이 시작할 수 있으면 이메일, Google Sheet, Notion 같은 수동 접수 흐름을 우선한다.
- 외부 연동, 결제, 고객 데이터 저장을 추가할 때는 환경변수와 접근권한을 먼저 설계한다.
- 사용자 입력은 경계에서 검증하고, 고객에게 보이는 오류는 내부 정보가 새지 않게 쓴다.
- secret, API key, service role key, 접근 토큰은 절대 커밋하지 않는다.

## 콘텐츠 원칙

- 권장 문구: `소유자 승인 기반 보안점검`, `웹앱 보안 설정 리뷰`, `AI 제작 웹사이트 출시 전 보안 체크`.
- 금지 문구: `해킹해드립니다`, `사이트 뚫어드립니다`, `무단 취약점 탐색`, `실서버 침투 테스트`.
- 가격은 초기 기준으로 Basic 99,000원, Standard 249,000원, Fix PR 149,000원부터를 기본안으로 둔다.
- 법률, 세무, 통신판매업, 개인정보 처리위탁은 유료 반복 운영 전 전문가 확인이 필요하다고 명시한다.

## 리포트 원칙

- 리포트는 점검 개요, 대상, 범위, 제외 범위, 요약, 발견 항목, 우선순위, 수정 가이드, 재점검 체크리스트, 면책을 포함한다.
- 발견 항목은 High / Medium / Low로 정렬한다.
- 각 finding에는 영향, 근거, 수정 권장, 우선순위를 적는다.
- 오탐 가능성은 숨기지 말고 검수 필요 상태로 표시한다.

## 작업 절차

- 비사소 작업은 `STATE.md`에서 현재 작업, 점수, profile, write set, contract freeze를 확인한 뒤 진행한다.
- `AGENTS.md` 자체는 100줄 이내로 유지한다.
- 구현 전에는 MVP 밖 기능인지 먼저 확인한다.
- 완료 전에는 가능한 repo 검증 명령과 문서 자체 검토를 수행한다.
- 브라우저, 배포 URL, 외부 도메인 smoke check는 사용자가 명시적으로 요청한 경우에만 한다.

## 오류 기록

- 작업에 영향을 주는 실행, 설치, 검증 오류는 `ERROR_LOG.md`에 append-only로 남긴다.
- 항목은 time, location, summary, details, status를 포함한다.
